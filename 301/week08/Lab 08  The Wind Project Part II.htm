Estimating the transition matrix

There are (as far as I know) no existing functions that quickly obtain transition matrices from a markov chain. Thankfully, it is relatively simple thing to do. Exercise: Refrain from looking below, and think about how.

getM.mc <- function(mc) {
    M.count <- as.matrix(table(mc[-length(mc)], mc[-1]))
    M.count/rowSums(M.count)

Defining a class

Ok, now for something new and powerful. R is an object oriented programming (OOP) style language in the sense that all objects in it are of a class, most in what is known as S3, some in the much stricter S4 and some in the mysterious to me R5. I'll talk about these (a bit) in lecture, but the important thing to know is that once an object is associated with a class, then you can define a method for several generic functions that is specific to thie class. Technically, the identification of the class dispatches the generic function to a class-specific function call. The most important and familar examples are: plot(), summary(), print(), all of which behave somewhat differently depending on the type of object they are applied to.

We can create this specific type of functionality in our package relatively simply. We would like to define two kinds of objects: a “Markov chain” (mc) class, and a “Transition Matric” (mtm) class, each of which will have its own plot() methods. Here are the steps:

To specify a class, you just use the class() function:

B1.mc <- getNESW(Theta[, 1])

Note that the mc class inherited the levels() attribute of its previous factor class. We now, simply, specify the class of the objects that our generating functions (getNESW() and getM()) return:

getM.mc <- function(mc) {
    M.count <- as.matrix(table(mc[-length(mc)], mc[-1]))
    M <- M.count/rowSums(M.count)
    class(M) <- "mtm"
    return(M)
}

getNESW <- function(Theta) {
    options(warn = -1)
    NESW <- cut(Theta, c(0, 45 + 90 * 0:3, 360), c("N", "E", "S", "W", "N"))
    NESW <- factor(as.character(NESW))
    NESW <- relevel(NESW, "N")
    class(NESW) <- "mc"
    return(NESW)
}

Simulating from transition matrix

To simulate a Markov chain from a matrix (following my lecture), you need only resample the next row with the appropriate probability distribution:

simChain <- function(M, t) {
    n <- nrow(M)
    Xs <- rep(0, t)
    X <- sample(1:n, 1)
    Xs[1] <- X
    for (i in 2:t) {
        Xs[i] <- sample(1:n, 1, prob = M[Xs[i - 1], ])
    }
    Chain <- factor(row.names(M)[Xs])
    class(Chain) <- "mc"
    return(Chain)
}

Simulation based inference

A natural question to ask about these chains is: Could the three chains' transition matrices actually be equal? We can use simulated chains to get confidence intervals around each of the transitions. , would be to (1) estimate M1, (2) recreate a large number N of simulated chains of length 365, (3) for each chain obtain the probability transition matrix, and (4) compute the 95% quantile range of the transitions.

Since we have all the pieces in place:

The average resimulation C.I.~is about 0.17, or an approximate s.d. of 0.0415. These results suggest that there is a fair amount of similarity among the matrices, but note all the CI's that don't overlap with the actual estimates! Note especially the S to E and the S to N transitions. This suggests strongly that the first order Markov model is not the best model for many of these transitions (or that there's an error in my code!)

Appendix

A likelihood based method for performing a test of a chain B corresponding to a transition matrix A is very succinctly summarized below (from this forum post: http://math.stackexchange.com/questions/86331/how-can-i-compare-two-matrices):

In other words, you want to know whether the process that you observed and which gave rise to B could correspond to a path of the Markov chain with transition matrix A, or not. A canonical way to answer this question is to compute the likelihood of the sequence of observations described by B with respect to the model A. For Markov chains, this is
L=∏x,yA(x,y)N(x,y),
where the product is over every states x and y and N(x,y) is the number of times one observed a transition from x to y. Presumably, B corresponds to B(x,y)=N(x,y)/N(x), where N(x) is the number of times one observed a visit to x.

If indeed the observed sequence was generated by the model A, N(x,y)=nm(x)A(x,y)+o(n) for every x and y, where n is the length of the sequene and m is the so-called stationary distribution of the Markov chain. Hence,

logLA=n⋅ΛA+o(n),ΛA=∑x,ym(x)A(x,y)logA(x,y).
Furthermore, the discrepancy between B and A under the hypothesis that the sequence was generated by A is described by a central limit theorem, that is, for every x and y,
N(x,y)=nm(x)A(x,y)+n−√Zx,y+o(n−√),
for a given centered gaussian family (Zx,y) whose covariance matrix depends on A and can be computed explicitly. Hence the approximate p-value of the observed sequence is
P(Z≈(N−nm⋅A)/n−√).
Assuming the covariance matrix of Z is C, one gets an approximate p-value of e−K/2n, with
K=(N−nm⋅A)∗C−1(N−nm⋅A).
One can also consider the likelihood L and note that
L=LnA⋅LB∣A,LB∣A=∏x,yA(x,y)N(x,y)−nm(x)A(x,y),
and use LB∣A to estimate the discrepancy between the observed sequence, described by (N(x,y)), and the model, described by A.
